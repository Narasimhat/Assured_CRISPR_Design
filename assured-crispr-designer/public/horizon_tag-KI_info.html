Horizon tag knock-in — design & documentation checklist

Standalone reference for tag / reporter HDR knock-in projects. Open this file in Microsoft Word and use Save As → .docx for an editable document. The same content is appended to C-terminal / N-terminal design reports downloaded from ASSURED CRISPR Designer.

This appendix summarizes conventional tag / reporter HDR knock-in documentation practice (including workflows similar to Horizon Pathfinder–style tag KI). Use it alongside the sequences and primers in the main report. Replace bracketed fields where your institution requires traceability (e.g. IRIS ID, GenBank accession).

1. Project identity & references

  • Official gene symbol, transcript / ENST (or RefSeq) used for amino-acid numbering.
  • Parental cell line, intended clonal line name, and internal project / IRIS identifier.
  • GenBank or assembly coordinates for the exact reference sequence used in this design package.
  • Cassette name and supplier or synthesis order reference for the tag / reporter block.

2. Knock-in architecture (C-terminal vs N-terminal)

  • C-terminal KI: confirm native stop codon removal / replacement; confirm new stop in donor; frame across 5′ HA → linker/tag → 3′ HA.
  • N-terminal KI: confirm ATG + Kozak context (if applicable); preserve reading frame after insert; confirm no unintended ATGs upstream of the intended start.
  • Document linkers, epitope tags (e.g. 2×HA), self-cleaving peptides (P2A/T2A), and fluorophores order.
  • State whether the final allele is fusion protein versus IRES / promoter-based reporter (if different cassette class).

3. Homology-directed repair (HDR) donor

  • Homology arm lengths (e.g. 250–800 bp per arm) matched to the uploaded reference locus.
  • Total donor length vs synthesis / cloning limits; fragmentation or vector strategy if > ~2–3 kb.
  • Strand convention for ordering (confirm order strand vs genomic / donor documentation).
  • Silent mutations in homology arms to disrupt PAMs / seed matches after successful HDR (when included).

4. gRNA & cutting strategy

  • SpCas9 (or other nuclease) format: RNP timing, concentration, and delivery method.
  • Cut site position relative to insertion junction; confirm micro-homology / MMEJ risk if critical.
  • Second gRNA or nickase strategy only if your protocol requires it (document rationale).

5. Ordering & traceability bundle

  • gRNA list (spacer, optional full sgRNA) aligned to vendor column map.
  • HDR donor: sequence checksum, yield / purification, and delivered format (ssDNA / dsDNA / plasmid).
  • Genotyping primers and predicted WT vs KI amplicon sizes (see main report tables).
  • Archive vendor quotations (PDF) with the design package; for Word documentation you can convert a quote PDF to HTML or paste tables into the main strategy report.
  • Optional: NGS panel / locus walk for clonal confirmation.

6. Experimental validation checklist

  • PCR across 5′ and 3′ junctions; Sanger traces archived with base calling.
  • Western / tag antibody / fluorescent protein confirmation as appropriate.
  • Off-target assessment per institutional SOP (if required).
  • Biosafety and GMO documentation, if applicable.

Disclaimer: This checklist is a project-documentation aid. It does not replace manufacturer instructions, regulatory filing requirements, or peer-reviewed protocols for your exact cell model.
